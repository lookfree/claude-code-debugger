# Claude Code 功能版本对照表 · harness-ide-spec 索引

这张表是 [claude-code-debugger 演进路径](../claude-code-debugger演进路径.md) 第一节那张大表的独立索引版。每个功能给一个稳定 ID，拆 issue 时直接引用（比如 "实现 OBS-01 session 监视器"）。

**列说明**——

- **ID**：稳定编号，按主题分段（ORCH 编排 / OBS 可观测 / HOOK / PERM 权限配置 / MODEL / SKILL / MISC），不随表格重排变化。
- **版本**：功能在 Claude Code 的引入/演进版本，已逐条对过官方 changelog（dream 与 Context Editing 是非 CLI 版本号，已标注）。
- **项目状态**：claude-code-debugger 当前对它的覆盖（无 / 部分 / 占位 / 只读配置）。
- **支柱**：归属五根产品支柱之一（配置 Cfg / 调试 Dbg / 观测 Obs / 编排 Cmp / 教学 Tch）。
- **Phase**：演进路径里落在哪个阶段。
- **Issue**：拆解状态占位（TODO / 草稿 / 已建 #N / 完成），后面手动维护。

校对基准日：2026-06-20。最新覆盖到官方 changelog 2.1.183 + dream（Research Preview）。

## 按 Phase 汇总（拆 issue 的执行顺序）

**Phase 0 · 止血**——SKILL-01（扫描路径，spec003）、MISC-11（依赖核验，spec022）、build 时序 bug（spec001）+ ENOENT 降级（spec002，无对应功能 ID，项目自身 bug）。

**Phase 1 · 补配置层**——ORCH-07、全部 HOOK-*、全部 PERM-*、全部 MODEL-*、SKILL-02~06、MISC-03/04/05，加 Agents 页真正实现。（MISC-10/12 移入暂不做。）

**Phase 2 · 运行时观测**——ORCH-01/02/04/05/06/08/09/10、OBS-01/02/03/06、MISC-06/07，加 hook 沙箱执行器（HOOK-01/02/04/06 的可执行化）。（OBS-04/05、MISC-08/13 移入暂不做。）

**Phase 3 · 编排与教学**——ORCH-03/09、MISC-01/02，加业务工作流模板 + Harness Benchmark + Onboarding Tour。

**暂不做**——MISC-09（Remote Control，跟 claudia 重叠）、MISC-10（LSP，配置面极小）、MISC-12（Windows/ARM64，平台适配是 Claude Code 自身的事）、MISC-13（Rewind，纯 TUI 交互无落盘）、OBS-04（Monitor，无独立落盘契约）、OBS-05（OTEL，走外部 collector 过重）、MISC-08（Memory Tool，API 侧能力）。详见 README「暂不做」小节的理由表。

## 维护约定

- 新版本 changelog 出来后，新功能在对应主题段追加一行，ID 顺延（不复用删除的号）。
- 项目状态随实现进度更新（无 → 部分 → 完成）。
- Issue 列建了 GitHub issue 后填 `#N`，合并后改 `完成`。
- 这张表只做"功能 → 落点"的索引；每个功能的详细 spec（数据源、UI、验收）写在 `harness-ide-spec/<ID>.md` 单独文件里，按需展开。
